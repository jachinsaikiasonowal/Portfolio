Case Study 06 · Founder Build ⚡ Founder

Building PromptOps — Git for LLM Prompts (0 → Beta)

SaaS Developer Tool · Solo Founder · January 2026 – Present

The Problem

Every AI engineering team I worked with had the same silent crisis: LLM prompts hardcoded into application logic with no version history, no testing pipeline, no rollback capability. One bad prompt change could break a customer-facing AI feature — and finding it took hours. I'd experienced this pain first-hand building my own agent workflows. No existing tool solved it cleanly. So I built it.

$0Starting budget — fully bootstrapped
SoloSole founder, designer, developer, marketer
CrowdedCompeting vs LangSmith, Langfuse, Humanloop
Jan 2026Zero to full beta in under 60 days
Product Built — Full Stack Solo 
# Before PromptOps — hardcoded, untracked
prompt = "Summarise this article in 3 bullets..."  # v??? in prod

# After PromptOps — versioned, tested, deployed in 10 seconds
from promptops import client
prompt = client.get("summarise-article", version="latest")
# rollback: client.rollback("summarise-article", steps=1)
  • Version control engine — every prompt commit stored with author, timestamp, message, and quality score. Full diff view. One-click rollback to any version in seconds, no deployment required
  • Prompt Studio — multi-model testing environment (GPT-4o, Claude, Gemini) with variable injection, live quality scoring, latency and cost visibility before any deployment
  • Automated eval suite — LLM-as-judge scoring + rule-based checks trigger on every commit. Bad prompts blocked before reaching production automatically
  • Instant deploy — PM edits a prompt in the dashboard, it's live in 10 seconds. Zero code deployment, zero engineering queue
  • Python SDK — single line replaces hardcoded prompt strings. Integration time under 30 minutes. JS/TS SDK in progress
  • Cost analytics dashboard — LLM spend broken down by prompt, model, team, and day with usage forecasting

10 Production AI Agents: Newsletter Ops on Autopilot

AI Systems Builder · AI News India · October 2024 – Present

The Challenge

Running a weekly AI newsletter solo meant 15–20 hours per week on research, writing, design, distribution, subscriber management, and analytics. The ceiling on quality was time. The solution wasn't to hire — it was to build a fleet of AI agents that handle the mechanical work so I could focus purely on editorial judgment and strategy.

20 hrsWeekly ops time before agents
SoloZero budget for staff
Phase 1n8n / Make.com workflow automations
Phase 2OpenClaw LLM agent architecture
Agent Portfolio — 10 Deployed in Production
  • Content Research Agent — monitors AI/tech sources daily, scores articles by relevance via LLM classification, surfaces top stories with pre-written summaries. Research time: 2 hours → 15 minutes
  • Newsletter Drafting Agent — multi-step pipeline (research → outline → first draft → self-edit) from approved story briefs. Writing time reduced 60% with human final-review gate
  • SEO Content Agent — generates long-form posts from keyword briefs using Chain-of-Thought prompting, auto-publishes to Ghost via Admin API, submits to Google Search Console for indexing
  • Subscriber Onboarding Agent — triggered on new signup: personalises welcome email, infers interest segment from source, applies Beehiiv tags, enrols into correct nurture track
  • Lead Scoring & Segmentation Agent — analyses engagement signals every 48 hours, dynamically updates segments across Beehiiv and ActiveCampaign
  • Product Promotion Agent — monitors lifecycle stage, triggers contextual product recommendation emails at optimal engagement windows. Conversion improved 40% vs static broadcasts
  • Social Distribution Agent — repurposes newsletter into platform-native formats for LinkedIn, X, and Medium. One editorial input → 7 content outputs, auto-scheduled
  • Customer Support Agent — handles tier-1 product enquiries, order status (Gumroad API), and refund requests autonomously. 90%+ queries resolved without human intervention
  • Weekly Analytics Agent — aggregates metrics from GA4, Beehiiv, and Gumroad, generates formatted report delivered to Notion every Monday morning
  • Performance Report Agent — weekly digest comparing current vs prior period across all channels, flags anomalies, and suggests 3 actions for the coming week
